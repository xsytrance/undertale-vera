    🧰 Tools & knowledge (especially if you can't run a local LLM)

    • You don't need an LLM at all. The most useful design idea here: every AI feature has a deterministic fallback — a plain, honest response built from parsed facts. An app like this stays fully functional with zero model. Build the fallback first; the model is garnish.
    • Ollama — the easiest local LLM runner. Install, then ollama pull llama3.1:8b, and you have a free model on your own machine (8GB+ RAM recommended).
    • Where saves live — Undertale: %LocalAppData%\UNDERTALE (Windows), ~/Library/Application Support/com.tobyfox.undertale (macOS), ~/.config/UNDERTALE (Linux). Deltarune: the same but DELTARUNE. Always copy your saves before experimenting.
    • Reading GameMaker saves — they're plain newline-delimited text + INI files. A text editor and patience is all it takes; the maps above give you a head start.
    • The evidence method — to map unknown fields: collect saves at labelled progression points, diff every line across the corpus, correlate against any summary file the game writes, and look for lines constant before an event and constant-but-different after it. Never guess.

    How It Works — from the simple version to the deep end

    You show it your save file. The characters talk to you about what's actually in it — your route, your LOVE, what you did — and they're never allowed to make those facts up. That's the whole trick. Everything below is just doing that honestly.

    Layer 1 — the pipeline ▾ 
    your save file → parser (read-only) → SaveTruth → the wall → a character's reply
                                          ↘ snapshots (your history, append-only)

    The parser reads the save and produces SaveTruth — a normalized record of what the file proves. The prompt shown to the AI is built from two buckets: SACRED (SaveTruth, stated verbatim, never overridable) and FREE (the character's voice, mood, lore). The motto over the whole machine: facts are sacred, feelings are free.

    Layer 2 — the wall, and staying honest ▾

    The two-bucket wall: personality may colour how a character speaks, never what the save says. Unreadable fields are null, never guessed. If the route is ambiguous, it's undetermined — the characters say "I can't tell yet" instead of guessing.

    Enforced, not promised: a hallucination guard re-checks every reply against SaveTruth after generation, and each reply carries a provenance chip (tap "✓ grounded") showing exactly which facts were sacred and which flavour was free.

    No model? Still works: every feature has a deterministic fallback built from the parsed facts alone, so the app never needs an LLM to be truthful — only to be charming.

    Layer 3 — where the field maps come from ▾

    The parser-truth law: a save field only gets a meaning when evidence earns it. Public documentation alone rates "medium". Promotion to "high" takes a real corpus: saves collected at labelled progression points, diffed line by line, cross-correlated against any summary file the game writes, and mined for partition flags — lines constant before an event and constant-but-different after it. That's how the Deltarune Jevil flag (line 559) was found. Everything unproven stays raw. The full maps are on The Commons page.

    Layer 4 — the live companion (Guided Mode) ▾

    Guided Mode never touches the game or its memory. A watcher polls the save folder; when the game writes a save and the file settles, it's re-parsed, a snapshot is appended, and the delta between saves becomes a beat ("kills rose from 0 to 3") that the characters react to. Hints are pre-written at three spoiler levels and gated by what the save proves about your progress — a hint never references content beyond where you actually are, and when the save shows too little, it says so.

    Layer 5 — patterns worth stealing ▾
    • Two buckets, one wall — separate what the model may colour from what it may never touch, and enforce it after generation, not just in the prompt.
    • Deterministic fallback first — design every AI feature to work with no model; add the model as voice, not as truth.
    • Evidence-promoted parsing — confidence tiers per field; corpora over guesses; honesty about layouts and versions.
    • Append-only history — snapshots never overwrite; the past stays true.
    • Progress-gated content — hints and lore keyed to proven progress, so nothing spoils.

    These patterns have a name and a family now — the introduction is one page over: 🌌 MultiVera.

    The MultiVera Project — one engine, many worlds

    A Vera is a companion app for a game you love: it reads your save file — read-only, evidence-first — and lets the game's own characters talk with you about your playthrough. Not generic chatbots wearing costumes: the facts come from your save and are never invented; the voices are original reinterpretations with real personality. Ember, the app you're in, is one Vera. It has siblings — and the whole idea is yours to take.

    🔥 Ember · Undertale + Deltarune Ch1

    The flagship — this app. Save parsing with corpus-earned field maps, route detection, grounded chat with seventeen characters across two worlds, Guided Mode (play beside it — it watches your save folder, never the game), an original CC BY soundtrack, and a capability ladder from zero-setup 🕯 Spark to fully local models.

    ⚔ FFT PSX Vera · Final Fantasy Tactics (PSX)

    The ancestor — where the campfire started. The same covenant pointed at a PlayStation-era tactics epic: your roster, your story progress, your war council. Ember inherited its architecture and then taught the engine new tricks.

    📡 MGS Vera · Metal Gear Solid

    The sketch — a deliberately tiny build: one character on a codec screen, no save parser at all, grounded in curated canon lore with provenance and ambiguity notes. Proof the idea scales down as well as up: a Vera can start as one honest voice.

    The covenant every Vera signs ▾

    Facts are sacred, feelings are free. A save field only gets a meaning when evidence earns it; unknowns stay null; ambiguity is said out loud (an unclear route is undetermined, never a guess). Personality may colour how a character speaks — never what the save says. And every feature works with no AI model at all, so the app never needs an LLM to be truthful — only to be charming. The deep dive is one page over: ❓ How It Works.

    Make your own Vera — the general recipe ▾

    You don't need permission and you don't need a big model. You need a game with readable saves and some love for its cast:

    • 1 · Read the save, read-only. A parser that never writes and never crashes — malformed input becomes warnings, not exceptions.
    • 2 · Earn every field. Collect real saves at known progression points and diff them; a field only gets a name when evidence corroborates it. (The Commons shows the maps this method produced for Undertale and Deltarune.)
    • 3 · Build the truth. Normalize into one record of what the file proves — unknowns null, confidence per field, moral forks honest.
    • 4 · Give the cast original voices. Reinterpretations in your own accent, never copied dialogue — and route-aware moods, so the world reacts to the player's choices.
    • 5 · Keep the wall. Facts from the save, flavour from the voice, a guard that re-checks every reply, and a scripted fallback so it all works model-free.
    • 6 · Then decorate. Hints, music, emblems, easter eggs — the parts that make it feel like a place instead of a tool.

    The full engineering spec — written from what the first packs actually required — lives in docs/GAME_PACKS.md.

    3 · The knobs — how to bend a prompt like these ▾
    • The role line — “You are X… never as an AI assistant.” One sentence buys the whole voice. Change the character, keep the shape.
    • The fact wall — facts go in a loud, fenced block (“HARD FACTS — NEVER OVERRIDE”), stated verbatim, with the reasons included so the model can explain without inventing. The block sits high: models weight early context.
    • Honesty valves — every unknown is named (“the file shows no kill count”) and the ask ends with “invent nothing beyond it.” An escape hatch beats a hallucination: give the model permission to say “I can't tell yet.”
    • Style dials — length (“one or two sentences”), register (“in your own voice”), format (“begin with a verdict line prefixed exactly with 'Verdict:'”). Hard formats get an exact token to anchor on.
    • Check after, not just ask before — the prompt promises honesty; a guard re-reads the reply against the facts and flags contradictions. Prompts persuade, checks enforce.
    • Design the fallback first — every feature works with no model at all (see 🕯 Spark mode). If your feature only works when the model behaves, it doesn't work.
    4 · The Suno prompts — how the whole soundtrack was asked for ▾

    The lesson learned first: the first prompt drafts said things like “chiptune, emotional, retro-RPG” — and came back generic. The fix was describing the DNA of the sound instead of gesturing at it: name the instruments (music box, felt piano, plucked SNES-style ostinato, choir “aah” pads), the tempo, the key, the feeling of a specific place — and always end with seamless loop, instrumental, no vocals. The finished prompts go further still — three more knobs: an Exclude list (a negative prompt naming everything the track must not become), a weirdness dial (how far from the style's center the model may drift) and a style strength dial (how hard the style prompt steers). The character themes below show the full draft → final journey, dials included. Every track is downloadable on The Commons (CC BY 4.0).

    Read a save

    Read-only — your save is never modified. Unknown fields stay null, never guessed.

    You need these 2 files from your Undertale save folder (either one alone works; both together read the most):
    • file0 — your save: LOVE, room, kills, flags
    • undertale.ini — name, play time, settings
    Where they're usually kept:
    • Windows %LocalAppData%\UNDERTALE\ (= C:\Users\you\AppData\Local\UNDERTALE)
    • macOS ~/Library/Application Support/com.tobyfox.undertale/
    • Linux ~/.config/UNDERTALE/
    • Steam Deck …/compatdata/391540/pfx/…/AppData/Local/UNDERTALE/
    ⚠ The files have no extension — file0 is literally "file0", not "file0.txt".
    🌑 Deltarune Chapter 1 works too: drop filech1_0 (from %LocalAppData%\DELTARUNE) into the file0 slot — Ember detects it by name and reads it as a Dark World save.
    🧭 Never done this before? Step-by-step ▾

    Two minutes, no tech skills needed. Your save is only read — the game never knows we were here.

    🪟 Windows ▾
    1. Press Win + R (the Windows key and R together).
    2. Paste this and press Enter: 
      %LocalAppData%\UNDERTALE
    3. A folder opens. Find file0 (no extension — that's normal).
    4. Back here: tap ① file0 below, pick that file, then Read save.
    5. Bonus: also pick undertale.ini in slot ② — Ember reads your name and play time from it.
    🍎 macOS ▾
    1. In Finder: menu Go → Go to Folder…
    2. Paste this and press Enter: 
      ~/Library/Application Support/com.tobyfox.undertale
    3. Find file0, then use slots ① and ② below and tap Read save.
    🐧 Linux / Steam Deck ▾
    1. Native: 
      ~/.config/UNDERTALE
    2. Steam/Proton (Deck included): 
      ~/.local/share/Steam/steamapps/compatdata/391540/pfx/drive_c/users/steamuser/AppData/Local/UNDERTALE
    3. Grab file0 (and undertale.ini if you like), load them below.
    🌑 Deltarune Chapter 1 ▾
    1. Same places, folder DELTARUNE instead of UNDERTALE.
    2. The file is 
      filech1_0
    3. Drop it in slot ① — Ember sees the name and reads it as a Dark World save.
